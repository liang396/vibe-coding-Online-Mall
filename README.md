# 二手商城

一个基于 **Spring Boot + React** 的二手交易平台，支持商品浏览、购物车、订单、评论、卖家管理，以及 **AI 问答助手**。

本项目的开发方式采用 **Vibe Coding**，整体思路参考了**程序员鱼皮**关于 Vibe Coding 项目开发流程的教程，并结合本仓库的实际开发过程持续迭代。

## 项目亮点

- 前后端分离，覆盖完整的二手交易基础链路
- 支持买家 / 卖家身份切换
- 商品、购物车、订单、评论形成闭环
- 新增 AI 问答模块，可结合商品详情和公开评价提供购买建议
- 对商品公开接口、缓存和鉴权链路做了稳定性修复

## 功能列表

### 用户与身份
- 用户注册、登录
- 个人资料查看与更新
- 买家 / 卖家身份切换
- 基于 Token 的登录态管理

### 商品与评论
- 商品列表分页浏览
- 关键词搜索、分类筛选
- 商品详情查看
- 卖家发布、编辑、删除商品
- 商品评论查看与提交

### 交易流程
- 加入购物车、修改数量、删除商品
- 创建订单
- 查看订单列表与订单详情
- 按买家 / 卖家角色控制订单状态流转

### AI 问答
- 独立 AI 问答页面
- 支持普通问答与商品上下文问答
- 可从商品详情页一键跳转到 AI 咨询
- 支持流式输出，逐步展示回复内容

## 项目结构

```text
.
├─ src/main/java/com/project        # 后端业务代码
├─ src/main/resources               # 配置与数据库脚本
│  └─ db                           # schema.sql / data.sql
├─ frontend/src                     # 前端源码
├─ pom.xml                          # 后端 Maven 配置
└─ frontend/package.json            # 前端依赖与脚本
```

## 主要页面

- 首页：`/`
- 商品列表：`/products`
- 商品详情：`/products/:productId`
- 购物车：`/cart`
- 订单页：`/orders`
- 用户中心：`/user`
- AI 问答：`/ai`

## 数据模型概览

当前数据库核心表包括：

- `users`
- `categories`
- `products`
- `orders`
- `order_items`
- `reviews`

其中：
- `users` 支持单账号多角色能力
- `products` 管理商品状态、库存、卖家归属
- `orders` 与 `order_items` 负责订单闭环
- `reviews` 提供商品公开评价数据，也会被 AI 模块复用

## 本地运行

### 1. 启动依赖服务
请先准备：
- MySQL
- Redis

### 2. 启动后端

```bash
mvn spring-boot:run
```

默认后端端口：`8080`

### 3. 启动前端

```bash
cd frontend
npm install
npm run dev
```

默认前端开发端口：`5173`

## 配置说明

项目中的数据库、Redis、AI 配置均支持通过环境变量覆盖。

常见配置包括：
- `DB_URL`
- `DB_USERNAME`
- `DB_PASSWORD`
- `REDIS_HOST`
- `REDIS_PORT`
- `REDIS_PASSWORD`
- `AI_ENABLED`
- `AI_BASE_URL`
- `AI_API_KEY`
- `AI_MODEL`

> 建议在本地或部署环境中使用环境变量，不要把真实密钥直接提交到仓库。

## AI 模块说明

AI 问答模块由后端代理第三方模型接口，前端不直接暴露模型密钥。

当前能力包括：
- 普通聊天问答
- 基于商品详情和评价的上下文问答
- 流式输出回复
- 基于 Redis 的轻量限流
- 统一错误提示与失败兜底

## 开发说明

这个项目不是单纯“让 AI 自由生成代码”，而是按 **Vibe Coding** 的方式逐步推进：
- 先明确需求边界
- 再梳理链路和数据流
- 每次只打通一个小闭环
- 通过日志、接口、页面一起验证
- 跑通后再继续迭代

整体方法参考了**程序员鱼皮**的 Vibe Coding 教程，并结合该项目在真实开发过程中遇到的登录、缓存、鉴权、订单、AI 接口等问题进行了落地调整。
